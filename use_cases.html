Діаграми діяльності

ID: SignIn
НАЗВА:Вхід до системи
УЧАСНИКИ:Користувач (гість), Система
ПЕРЕДУМОВИ:Користувач має зареєстрований обліковий запис
РЕЗУЛЬТАТ:Користувач успішно увійшов до системи
ВИКЛЮЧНІ СИТУАЦІЇ:Некоректна електронна пошта — IncorrectEmail
Некоректний пароль — IncorrectPassword
ОСНОВНИЙ СЦЕНАРІЙ:1. Користувач натискає кнопку «Увійти» на головній сторінці.
2. Вводить електронну пошту та пароль у форму входу.
3. Натискає кнопку «Підтвердити вхід».
4. Система перевіряє облікові дані.
5. Система повідомляє про успішний вхід.

ID: SignUp
НАЗВА:Реєстрація в системі
УЧАСНИКИ:Користувач (гість), Система
ПЕРЕДУМОВИ:Користувач не має зареєстрованого облікового запису
РЕЗУЛЬТАТ:Обліковий запис користувача створено
ВИКЛЮЧНІ СИТУАЦІЇ:Пароль не відповідає вимогам — InvalidPassword
Вказана електронна пошта недійсна — InvalidEmail
ОСНОВНИЙ СЦЕНАРІЙ:1. Користувач натискає кнопку «Зареєструватися» на головній сторінці.
2. Вводить електронну пошту та пароль у форму реєстрації.
3. Натискає кнопку «Підтвердити реєстрацію».
4. Система перевіряє введені дані.
5. Система надсилає лист-підтвердження на вказану електронну пошту.
6. Користувач підтверджує реєстрацію за посиланням у листі.
7. Система створює обліковий запис.
8. Система повідомляє про успішну реєстрацію.

ID: Logout
НАЗВА:Вихід з системи
УЧАСНИКИ:Користувач (гість), Система
ПЕРЕДУМОВИ:Користувач авторизований в системі
РЕЗУЛЬТАТ:Виконано вихід з облікового запису користувача
ВИКЛЮЧНІ СИТУАЦІЇ:Відсутні
ОСНОВНИЙ СЦЕНАРІЙ:1. Користувач натискає кнопку "Вийти" на головній сторінці.
2. Користувач натискає кнопку "Підтвердити".
3. Система завершує сеанс користувача.
4. Сиситема повідомляє про успішний вихід з облікового запису.

ID: Search
НАЗВА:Пошук даних за запитом
УЧАСНИКИ:Користувач, Система
ПЕРЕДУМОВИ:У системі наявна база даних для пошуку
РЕЗУЛЬТАТ:Відібрані за запитом дані відображено користувачу
ВИКЛЮЧНІ СИТУАЦІЇ:Дані не знайдено — DataNotFound
ОСНОВНИЙ СЦЕНАРІЙ:1. Користувач натискає кнопку «Пошук» на головній сторінці.
2. Вводить запит у пошуковий рядок.
3. Натискає кнопку «Знайти».
4. Система виконує пошук за введеним запитом.
5. Система виводить результати пошуку користувачу.

ID: Download
НАЗВА:Завантажити дані з системи
УЧАСНИКИ:Користувач, Система
ПЕРЕДУМОВИ:Користувач авторизований у системі
РЕЗУЛЬТАТ:Дані завантажено на пристрій користувача

Даних не існує - DataDoesNotExist

3. Користувач натискає кнопку "Підтвердити".
4. Система створює файл з даними.
5. Система завантажує дані на пристрій користувача.
6. Система повідомляє, що дані успішно завантажено.

ID: BanUser
НАЗВА:Заблокувати користувача
УЧАСНИКИ:Адміністратор, Система
ПЕРЕДУМОВИ:Адміністратор авторизований
РЕЗУЛЬТАТ:Обліковий запис користувача заблоковано
ВИКЛЮЧНІ СИТУАЦІЇ:Користувача не існує - UserNotFound
Користувач вже заблокований - UserAlreadyBanned
ОСНОВНИЙ СЦЕНАРІЙ:1. Адміністратор натискає кнопку "Управління користувачами" на головній сторінці.
2. Адміністратор вводить ім'я користувача в пошуковий рядок.
3. Адміністратор натискає кнопку "Знайти".
4. Система здіснює пошук за запитом.
5. Система відображає шуканого користувача.
6. Адміністратор натискає кнопку "Заблокувати".
7. Адміністратор виставляє таймер блокування.
8. Адміністратор натискає кнопку "Підтвердити".
9. Система блокує користувача на вказаний період.
10. Система повідомляє, що користувача успішно заблоковано.
